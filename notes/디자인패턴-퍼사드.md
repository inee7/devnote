# 퍼사드 패턴

## 한 줄 요약

복잡한 구현 세부 사항을 감추고 간단한 인터페이스를 제공하는 패턴

## 핵심 정리

- 지저분한 구현 세부 사항을 감추는 데 사용
- 목적은 서로 연관된 여러 클래스나 인터페이스를 더 깔끔하고 간단하게 다룰 수 있도록 하는 것
- 코틀린에서는 확장 함수로도 간단히 퍼사드를 표현할 수 있음

## 상세 내용

### 코틀린에서의 구현

자바에서는 클래스를 별도로 만들고 모든 로직을 캡슐화한 함수를 만든다. 코틀린에서는 확장 함수로도 간단히 퍼사드를 표현할 수 있다.

```kotlin
fun Server.startFromConfiguration(fileLocation: String) {
    val path = Path(fileLocation)

    val lines = path.toFile().readLines()

    val configuration = try {
        JsonParser().server(lines)
    } catch (e: RuntimeException) {
        YamlParser().server(lines)
    }

    Server.withPort(configuration.port)
}

class Server private constructor(port: Int) {
    companion object {
        fun withPort(port: Int): Server {
            return Server(port)
        }
    }
}
```

### 특징

- 확장 함수를 활용하여 기존 클래스에 복잡한 로직을 캡슐화
- 여러 단계의 복잡한 처리를 하나의 함수로 감춤
- 클라이언트 코드는 단순한 인터페이스만 사용
