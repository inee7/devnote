# 빌더 패턴

## 한 줄 요약

복잡한 객체 생성 과정을 단계별로 구성하는 패턴 (단, 코틀린에서는 기본 매개변수와 명명 인수로 대체 가능)

## 핵심 정리

- 기본 매개변수와 명명 인수가 있으면 빌더 패턴을 만들 필요 없음
- 코틀린의 언어 기능이 빌더 패턴의 목적을 자연스럽게 해결

## 상세 내용

### 코틀린에서의 구현

```kotlin
class Mail internal constructor(
    val to: List<String>,
    val cc: List<String>? = listOf(),
    val title: String? = "",
    val message: String? = "",
    val important: Boolean = false
)

val mail = Mail(title = "안녕", message = "잘지내니?")
```

### 특징

- `internal constructor`: 생성자를 모듈 내부에서만 접근 가능하도록 제한
- 기본 매개변수: 선택적 파라미터를 기본값으로 설정
- 명명 인수: 필요한 파라미터만 지정하여 객체 생성
- 전통적인 빌더 패턴의 장황함을 제거

### 언제 사용하지 않는가

코틀린에서는 기본 매개변수와 명명 인수가 있기 때문에, 대부분의 경우 별도의 빌더 클래스를 만들 필요가 없다.
