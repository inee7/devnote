# 싱글톤 패턴

## 한 줄 요약

애플리케이션 전체에서 단 하나의 인스턴스만 존재하도록 보장하는 패턴

## 핵심 정리

- `object` 키워드로 쉽게 싱글톤 객체를 생성 가능
- 직접 인스턴스화할 수 없기에 생성자는 정의하지 않음
- 클래스와 인터페이스 상속이 가능
- 접근했을 때만 실행하는 lazy 초기화

## 상세 내용

### 기본 사용법

```kotlin
object NoMoviesList
```

### 인터페이스 상속

```kotlin
internal object EmptyList : List<Nothing>, Serializable, RandomAccess {
    // ...
}
```

### 초기화

접근했을 때만 실행하는 lazy 초기화이다.

```kotlin
object Logger {
    init {
        println("I was accessed for the first time")
        // Initialization logic goes here
    }

    fun log(message: String) {
        println("Logging $message")
    }
    // More code goes here
}
```

### 특징

- `object` 키워드 하나로 싱글톤 구현
- Thread-safe한 lazy 초기화 자동 제공
- 직접 인스턴스화 불가능
- 클래스, 인터페이스 상속 가능
- 첫 접근 시점에 초기화됨
