# 코틀린에서 제약 처리 
```kotlin
require(num <= size) { //아규먼트 제약
	"error message"
}

check(isOpen) { "error message"} // 상태 제약 

return // 제약을 만족 못하면 

?: return //제약을 만족 못하면
?: throw
```
이렇게 require, check, return, ?: 등으로 조건을 제한할 수 있는데 이를통해
* 문서를 읽지 않아도 문제를 확인 가능 
* 예외가 터져 코드 안정적 
* 단위테스트를 줄일수 있음 
* 스마트 캐스트 기능할 활용할 수 있게 되어 캐스트를 적게 할 수 있음 

require -> IllegalArgumentException
check -> IllegalStateException

스마트캐스팅
```kotlin
fun changeDress(person:Person) {
	require(person.outfit is Dress) // 여기서 스마트 캐스팅이 됨 
	val dress: Dress = person.outfit
```
```kotlin
require(person.email != null) // 이후 코드에서 email이 스마트 캐스팅이 되어 notnull타입으로 쓸수 있다 
requireNotNull(person.email) // 더 쉽게 
```

null 안전하게 처리 
```kotlin
val email: String = person.email ?: return
val email: String = person.email ?: run { //로그 남기고 싶을때 
	log("Email not sent, no email address")
	return 
}
```
return과 throw를 활용한 elvis는 nullable 확인할 때 굉장히 많이 사용
