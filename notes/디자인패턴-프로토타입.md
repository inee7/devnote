# 프로토타입 패턴

## 한 줄 요약

기존 객체를 복사하여 일부만 다른 새로운 객체를 만드는 패턴

## 핵심 정리

- 객체의 일부만 다른 또 다른 객체를 만들 때 사용
- 코틀린의 `copy()` 함수로 프로토타입 패턴을 나타낼 수 있음
- 데이터 클래스에서 자동으로 제공되는 기능

## 상세 내용

### 데코레이터 패턴과의 차이

- 프로토타입 패턴: 속성을 조금씩 다르게 여럿 만들 수 있음
- 데코레이터 패턴: 동작을 조금씩 다르게 여럿 만들 수 있음

### 코틀린에서의 구현

```kotlin
data class User(
    val name: String,
    val role: Role,
    val permissions: Set<String>,
)
val prototype = u.copy(name = _name)
```

### 특징

- `data class`를 사용하면 `copy()` 메서드가 자동 생성됨
- 명명된 인자를 사용하여 변경할 속성만 지정
- 불변 객체를 사용하는 함수형 프로그래밍 스타일과 잘 맞음
