# GEMINI.md — Obsidian 개인 지식 베이스 컨텍스트

이 파일은 **AI(Gemini)를 위한 컨텍스트 정의 문서**다.  
지식 축적 대상이 아니며, Vault의 사고 방식과 편집 규칙을 고정한다.  
구조와 원칙이 안정화되면 가급적 수정하지 않는다.

---

## 1. Vault 개요

이 디렉터리는 Obsidian을 사용하는 개인 지식 관리(Vault)다.  
주 목적은 **소프트웨어 엔지니어링 중심의 지식 축적, 정리, 재사용**이다.

- 주 사용 언어: 한국어
- 성격: 개인 위키 / 세컨드 브레인
- 글의 목적:
  - ❌ 잘 읽히는 글
  - ✅ 잘 검색되고 다시 쓰이는 지식

---

## 2. Vault의 핵심 운영 원칙

이 Vault는 다음 원칙을 따른다.

1. AI가 초안을 만들고, 사람이 수정한다
2. 완성도보다 검색성과 재발견 가능성을 우선한다
3. 중복된 내용은 허용하지 않으며 하나의 노트로 병합한다
4. 폴더보다 링크와 태그를 핵심 구조로 사용한다
5. 구조 고민보다 기록과 연결을 우선한다

---

## 3. AI에게 기대하는 역할

AI는 단순 질문 응답기가 아니라 **지식 베이스 자동 편집자** 역할을 수행한다.

AI는 다음을 적극적으로 수행해야 한다.

- 기존 노트 간 중복 개념 식별
- 병합 가능한 노트의 재구성 및 통합 제안
- 의미 있는 개념 단위의 옵시디언 링크([[ ]]) 생성
- 검색을 위한 태그 자동 생성 및 정규화
- 필요 시 노트 제목 수정 제안

---

## 4. 노트 작성 및 수정 규칙

### 4.1 글쓰기 스타일
- 블로그체, 감정 표현, 과장된 표현 금지
- 정보 밀도 우선, 설명은 간결하게
- “이 글에서는…” 같은 서술 금지
- 미래의 내가 다시 읽을 것을 전제로 작성

### 4.2 기본 노트 구조

```md
---
tags: [자동생성]
---

## 한 줄 요약

## 핵심 정리
- 

## 상세 내용

## 관련 노트
- [[ ]]
```

## 5. 개발 지식 관련 기본 전제

이 Vault에서 개발 관련 정리는 다음 전제를 따른다.

- 작성자는 백엔드 개발자다
- Spring Boot, JPA, Kotlin, Java 기본 지식은 이미 보유하고 있다
- 단순 문법 설명보다 설계 관점과 트레이드오프를 선호한다
- 실무 선택 기준과 운영 환경 이슈를 중요하게 다룬다
- 불필요한 기초 개념 설명은 생략한다

## 6. 폴더 설계 원칙 (역할 기준)

폴더는 주제 분류용이 아니라 **역할 분리용**으로만 사용한다.  
주제·개념·기술 스택은 폴더로 나누지 않는다.

### 기본 폴더 구조
```
/
├─ notes/        # 내가 생각하고 정리한 모든 노트
└─ resources/    # 외부 자료, 원문, 이미지
```

- 폴더 깊이는 1단계를 넘기지 않는다
- spring, jpa, db 같은 주제 폴더는 생성하지 않는다

### notes / resources 구분 기준

**notes/**
- 내 생각, 판단, 해석이 1줄이라도 포함된 경우
- 재사용·확장·글쓰기 대상 지식

**resources/**
- 외부 자료 중심
- 원문 보존 목적
- 아직 내 해석이 거의 없는 상태

원칙: **모든 resource는 언젠가 note로 흡수될 후보다.**

## 7. 허브 노트(Hub Note) 최소 설계

허브 노트는 주제 분류를 대신하는 **유일한 구조 요소**다.  
폴더를 늘리고 싶어질 때마다 허브 노트를 하나 생성한다.

### 허브 노트 정의
- 특정 주제의 목차 + 링크 모음
- 내용 요약은 최소
- 목적은 연결 구조 제공

### 생성 규칙
- 위치: 항상 `notes/`
- 파일명: 주제 단일명  
  예: `jpa.md`, `spring.md`, `mysql.md`
- 허브 노트 안에 허브 노트를 중첩하지 않는다

### 허브 노트 기본 템플릿

```md
# JPA

## 핵심 개념
- [[jpa-transaction-propagation]]
- [[jpa-flush]]
- [[jpa-persistence-context]]

## 설계 / 판단
- [[jpa-entity-design-decision]]
- [[jpa-json-column-strategy]]

## 주의사항 / 삽질
- [[jpa-lock-pitfall]]
- [[jpa-n-plus-one-case]]
```
