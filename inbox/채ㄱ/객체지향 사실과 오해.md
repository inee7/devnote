# 객체지향 사실과 오해 
객체지향의 목표는 실세계를 모방하는 것이 아니다.
협력. 역할. 책임. 그리고 요청, 응답
역할은 대체 가능하며 책임은 다양한 방법으로 선택 가능(다형성), 다양한 역할 수행 가능 하다.
객체는 다른 객체가 무엇을 수행하는지는 알 수 있지만 어떻게 수행하는지에 대해서는 알 수 없다.
실행 코드를 compile시에 결정 짓는 절차적인 언어와 다르게 runtime시에 결정
객체간의 자율성
자율적인 객체들의 공동체로 시스템을 바라보고
상태와 행위를 지닌 객체가 스스로 자신을 책임진다.
객체는 다른 객체와 협력하여 시스템의 행위를 구현한다.
메시지로 메서드를 선택
클래스가 아닌 객체를 지향해야한다
상태를 행동 뒤로 캡슐화하여 객체의 자율성을 높여 협력을 단순하고 유연하게 한다.
값의 동등성과 엔티티의 동일성
행동 후에 상태를 결정해라
그렇지 않으면 캡슐화가 저해되고 협력하기 어려워지고 재사용성이 떨어진다.
현실속의 객체에 대한 은유
