민첩성
초당 배포
빠른 배포주기는 민첩성을 나타내
인프라 구축시간 단축한 클라우드 덕에 비지니스 개발의 민첩성을 촉진
서비스를 분리해서 스케일업아웃
서비스별 기술
기능하향식분해하는 구조적→ 객체로 모듈화 객채지향→ 기능별로 모듈화 CBD → 비지니스별 모듈화 SOA
SOA랑 MSA는 여러개 응집된 비지니스 서비스들로 개발한다는점으로 비슷한데
SOA는 구체적이지 않고 이론적이며 비지니스 성공사례가 없다.
클라우드 인프라의 등장으로 하드웨어를 유연하게 다룰수 있게 되면서 성공적인 시스템 구조인 MSA가 됨
각 서비스는 개별 프로세스에서 실행되고
HTTP자원 API 같은 가벼운 수단을 사용해서 통신
한서비스는 비지니스 기능 단위.
자동화된 배포 방식을 이용해 독립적 배포.
각 서비스 독립된 데이타와 기술 사용
MSA의 차이점
서비스별 저장소를 분리해서 다른 서비스가 저장소를 직접 호출하지 못하게 캡슐화 . API 를 통해서만.
REST API과 같은 가벼운 개방형 표준을 사용해 서비스간 느슨한 연계로 누구나 쉽게 사용 (SOA에서는 SOA프로토콜과 XML)
다기능팀으로 피자두판 정도 규모
각 서비스팀에 맞게 하는걸 폴리그랏 프로그래밍
