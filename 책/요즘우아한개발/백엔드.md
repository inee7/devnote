# 백엔드
입력값 검증 필수로 하자

어드민에서 다양한 검색은 query dsl을 사용
조건 파라미터중 둘중 하나만 오는 케이스에는 특히 검증 해야함

**분산캐시를 쓰려면 성능 테스트 필수!!**
레디스나 멤캐시에서 너무 많이 데이터를 담으면 히트율 높아져도 데이터가 네트워크 대역폭을 잡아먹어서 느려짐 
실제 운영 서비스를 기준으로 성능테스트를 하고 충분한 대역폭이 확보되어 있는지 확인 필요 (통신마다 대역폭 할당을 받는데 통신이 매우 잦으면 대역폭 할당이 많아져 성능저하를 줘)

파일크기가 큰 경우 캐시에 대규모이벤크 발생시 대역폭 많이 차지하고 성능 저하

캐시필드 조심히 다뤄야해

**인증된 후 권한 검사도 하자**
api를 인증처리를 했으나 유저와 api해당하는 리소스에 대해 검사를 하지 않으면 유저정보가 뚫릴수 있다 

로컬에서 운영프로필로 애플리케리션 띄워서 운영 MQ빨아들인 사연

API로 데이터 자주 변경한다면 어드민 만들어 인증인가 부여

배치 동작 확인은 그라파나로 (혹은 influxdb)
