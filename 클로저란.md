# 클로저란
클로저(Closure)는 함수형 프로그래밍에서 중요한 개념 중 하나로, 함수가 선언될 때의 환경을 기억하고, 그 함수가 선언된 위치의 스코프에 있는 변수에 접근할 수 있도록 하는 특성을 나타냅니다. 클로저는 자신을 포함하는 외부 함수의 변수에 접근할 수 있는 함수를 말합니다.

클로저는 크게 두 가지 주요 특성을 갖습니다:

1. **변수 캡처 (Variable Capture):** 클로저는 자신을 감싸고 있는 외부 함수의 변수를 캡처하고 저장할 수 있습니다. 이는 외부 변수의 생명 주기가 종료되더라도 클로저가 여전히 해당 변수에 접근할 수 있음을 의미합니다.

2. **환경 유지 (Environment Retention):** 클로저는 자신이 선언된 위치의 환경을 기억합니다. 이는 클로저가 선언된 스코프의 변수를 계속 참조하며, 이 스코프의 상태를 변경할 수 있음을 의미합니다.

간단한 클로저 예제를 통해 설명하겠습니다. 아래의 JavaScript 코드에서 `outerFunction`은 클로저를 생성하고 반환합니다. 이 클로저 `innerFunction`은 `outerFunction`의 변수인 `x`에 접근할 수 있습니다.

```javascript
function outerFunction() {
  let x = 10;

  function innerFunction() {
    console.log(x); // outerFunction의 변수에 접근
  }

  return innerFunction;
}

const closure = outerFunction();
closure(); // 출력 결과: 10
```

이 예제에서 `innerFunction`은 `outerFunction`의 변수 `x`에 접근할 수 있으며, `outerFunction`이 호출될 때의 환경을 기억합니다. 클로저를 이용하면 함수형 프로그래밍에서 상태를 보존하고 함수 간에 데이터를 공유할 수 있습니다.
